# Threat Model: Multi-Cloud Node Anomaly Detection

This threat model outlines the six core node-level anomalies monitored by the Multi-Cloud Node Anomaly Detection System across AWS (VPC, IAM) and Azure (VNet, Azure AD).

| Threat Type | Description | Attacker Capability | Operational & Business Impact |
| :--- | :--- | :--- | :--- |
| **1. Compromised VM** | Unexpected process execution, privilege escalation (e.g., sudo abuse, exploit scripts) on a virtual machine. | **Medium**: Gained remote code execution (RCE) via application-layer vulnerability, exposed SSH keys, or leaked credentials. | **High**: Full control over host operating system, access to local files/secrets, and a launchpad for further network reconnaissance. |
| **2. Malicious Container** | Unusual container behavior, unexpected binary execution, or container escape attempts (breakout to host kernel). | **Medium-High**: Exploited application vulnerabilities within a container or injected compromised images into public/private registries. | **High**: Host kernel access, container escape to compromise adjacent workloads, unauthorized access to Kubernetes API, or resource hijacking. |
| **3. Insider Misuse** | Abnormal access patterns by valid credentials, off-hours access, or unusual data reads from unfamiliar locations. | **High**: Legitimately assigned user or service principal credentials with native network access and infrastructure familiarity. | **Critical**: Unauthorized data exfiltration, deletion of production backups/resources, privilege abuse, and bypass of standard edge security controls. |
| **4. Botnet Node** | Periodic outbound Command and Control (C2) beaconing, outbound scanning, or data exfiltration via DNS tunneling. | **Medium**: Gained persistence on the node, installed beaconing scripts, and established communication channels with external domains. | **Medium-High**: Egress bandwidth consumption, inclusion of corporate IP addresses in global blacklists, data exfiltration, or resource hijacking (e.g., cryptomining). |
| **5. Misconfigured Node** | Open high-risk ports (e.g., 22/3389 to 0.0.0.0/0), overly permissive IAM/Azure role policies, public exposure. | **Low**: Can scan public IP ranges to discover exposed services, or exploit overly broad default permissions. | **High**: Egress/ingress security policy violations, credential harvesting from instance metadata endpoints, and unauthorized service modifications. |
| **6. Lateral Movement** | Unusual east-west network traffic, SSH/RDP sweeping, or RPC calls from a compromised node to other nodes. | **Medium**: Gained an initial foothold on a single subnet node, allowing network sniffing and scanning of adjacent internal systems. | **Critical**: Spread of compromise to high-value internal systems (e.g., databases, key vaults) which are not directly exposed to the internet. |
